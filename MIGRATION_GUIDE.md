# Миграция с localStorage на Firebase

Если вы ранее использовали версию приложения с localStorage, эта инструкция поможет вам сохранить ваши данные.

## Что изменилось?

### Теперь используется Firebase:
- ✅ **Аутентификация** - Firebase Authentication (вместо localStorage mock)
- ✅ **Комментарии** - Firestore (вместо localStorage)
- ✅ **Избранное** - Firestore (новая функция!)
- ⚠️ **Новости** - всё ещё localStorage (для простоты)

## Шаг 1: Настройка Firebase

Следуйте инструкции в [FIREBASE_SETUP.md](./FIREBASE_SETUP.md) для настройки Firebase проекта.

## Шаг 2: Миграция пользователей

### Сохранение старых данных

Ваши старые пользователи хранились в `localStorage` под ключом `mockUsers`. Вам нужно:

1. **Зарегистрироваться заново** в новой системе Firebase
2. Если вы были первым пользователем (админом), зарегистрируйтесь первым снова

### Важно!
⚠️ **Старые пароли не будут работать** - Firebase использует другую систему аутентификации.

## Шаг 3: Миграция комментариев

### Экспорт комментариев из localStorage

Если у вас есть важные комментарии:

1. Откройте консоль браузера (F12)
2. Выполните:
```javascript
const oldComments = JSON.parse(localStorage.getItem('mockComments') || '[]');
console.log(JSON.stringify(oldComments, null, 2));
```
3. Скопируйте результат

### Импорт в Firestore (вручную)

К сожалению, автоматической миграции комментариев нет. Вам нужно:
- Либо заново создать важные комментарии после авторизации
- Либо написать скрипт миграции (см. ниже)

### Скрипт миграции комментариев (для разработчиков)

Создайте файл `migrate-comments.js` и выполните его один раз:

```javascript
import { collection, addDoc } from "firebase/firestore";
import { db } from "./src/lib/firebase";

async function migrateComments() {
  const oldComments = JSON.parse(localStorage.getItem('mockComments') || '[]');
  
  for (const comment of oldComments) {
    try {
      await addDoc(collection(db, "comments"), {
        newsId: comment.newsId,
        author: comment.author,
        authorId: comment.authorId, // Замените на новый Firebase UID
        text: comment.text,
        createdAt: comment.createdAt,
      });
      console.log('Migrated:', comment.id);
    } catch (error) {
      console.error('Error migrating comment:', comment.id, error);
    }
  }
  
  console.log('Migration complete!');
}

migrateComments();
```

**Примечание:** Вам нужно будет заменить `authorId` на новые Firebase UIDs.

## Шаг 4: Новости (не требует миграции)

Новости остались в localStorage, так что они не изменятся после обновления.

## Шаг 5: Очистка старых данных (опционально)

После успешной миграции вы можете очистить старые данные:

```javascript
// В консоли браузера (F12)
localStorage.removeItem('mockUsers');
localStorage.removeItem('mockCurrentUser');
localStorage.removeItem('mockComments');
```

⚠️ **Внимание:** Не удаляйте `mockNews` - там хранятся ваши новости!

## Новые возможности после миграции

### 1. Избранные новости ❤️
Теперь вы можете:
- Добавлять новости в избранное
- Просматривать избранное на отдельной странице
- Синхронизировать избранное через Firestore

### 2. Улучшенная система комментариев
- Комментарии синхронизируются через Firestore
- Более надежное хранение
- Подготовка к реал-тайм обновлениям

### 3. Настоящая аутентификация
- Безопасная аутентификация через Firebase
- Возможность сброса пароля (при настройке)
- Готовность к социальным логинам (Google, Facebook и др.)

## Решение проблем

### "Не могу войти со старым паролем"
➡️ Это ожидаемо. Зарегистрируйтесь заново с тем же email.

### "Мои комментарии исчезли"
➡️ Старые комментарии из localStorage не мигрируются автоматически. Используйте скрипт миграции или создайте их заново.

### "Новости пропали"
➡️ Новости остались в localStorage. Проверьте, что вы не очистили кэш браузера.

### "Я был админом, теперь нет"
➡️ Зарегистрируйтесь первым в новой системе, чтобы снова стать администратором.

## Альтернативный подход: Чистый старт

Если миграция кажется сложной, вы можете:
1. Экспортировать важные новости (скопировать `mockNews` из localStorage)
2. Очистить localStorage полностью
3. Настроить Firebase с нуля
4. Зарегистрироваться как новый пользователь
5. Создать новости заново через админ-панель

## Поддержка

Если у вас возникли проблемы с миграцией:
1. Проверьте консоль браузера на наличие ошибок
2. Убедитесь, что Firebase правильно настроен
3. Создайте issue на GitHub с описанием проблемы
