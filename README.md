# NewsHub - Новостной портал

Современный новостной сайт с системой авторизации Firebase, комментариями в Firestore и административной панелью.

## 🚀 Быстрый старт

### Основной функционал:

✅ **Firebase Authentication** - регистрация и авторизация через email/пароль
✅ **Firestore Database** - хранение комментариев и избранных новостей
✅ **Система ролей** - первый пользователь становится админом
✅ **CRUD новостей** - для администраторов
✅ **Комментарии** - сохраняются в Firestore
✅ **Избранные новости** - добавление в избранное с синхронизацией в Firestore
✅ **Фильтрация по категориям** - технологии, бизнес, спорт и др.
✅ **Автоматические демо-новости** - при первом запуске

## 📦 Установка

```bash
npm install
```

## 🔧 Настройка Firebase

**Обязательно!** Для работы приложения необходимо настроить Firebase:

1. Создайте проект в [Firebase Console](https://console.firebase.google.com/)
2. Включите **Authentication** (Email/Password)
3. Создайте **Firestore Database**
4. Создайте файл `.env` в корне проекта:

```env
VITE_FIREBASE_API_KEY=your_api_key_here
VITE_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_project.appspot.com
VITE_FIREBASE_MESSAGING_SENDER_ID=your_sender_id
VITE_FIREBASE_APP_ID=your_app_id
```

📚 **Подробная инструкция:** см. [FIREBASE_SETUP.md](./FIREBASE_SETUP.md)

📊 **Настройка индексов:** см. [FIRESTORE_INDEXES.md](./FIRESTORE_INDEXES.md)

## 🗂️ Структура данных

### Firebase Authentication
Аутентификация пользователей через email/пароль

### Firestore Collections

#### `users`
```javascript
{
  uid: string,              // ID пользователя из Firebase Auth
  email: string,           // Email пользователя
  displayName: string,     // Отображаемое имя
  role: "admin" | "user",  // Роль пользователя
  createdAt: string        // Дата создания
}
```

#### `comments`
```javascript
{
  id: string,           // Автоматический ID документа
  newsId: string,       // ID новости
  author: string,       // Имя автора
  authorId: string,     // ID автора (из Firebase Auth)
  text: string,         // Текст комментария
  createdAt: string     // Дата создания
}
```

#### `favorites`
```javascript
{
  id: string,          // Автоматический ID документа
  userId: string,      // ID пользователя
  newsId: string,      // ID новости
  addedAt: string      // Дата добавления
}
```

### localStorage
Новости хранятся в localStorage для упрощения демонстрации (можно мигрировать в Firestore при необходимости)

## 💡 Важные заметки

- **Первый пользователь = Администратор** - зарегистрируйтесь первым для полного доступа
- **Firebase обязателен** - без настройки Firebase приложение не будет работать
- **Демо-данные** - при первом запуске создаются примеры новостей в localStorage
- **Безопасность** - настройте правила Firestore для продакшена (см. FIREBASE_SETUP.md)
- **Индексы** - Firestore автоматически предложит создать необходимые индексы

## 🔐 Безопасность

Правила безопасности Firestore настроены так, что:
- Все могут читать комментарии
- Только авторизованные пользователи могут создавать комментарии
- Пользователи могут удалять только свои комментарии
- Администраторы могут удалять любые комментарии
- Избранное доступно только владельцу

## 🚀 Деплой

Для продакшена:
1. Настройте Firebase правила безопасности
2. Добавьте домен в Firebase Authentication
3. Создайте production build: `npm run build`
4. Разверните на хостинге (Vercel, Netlify, Firebase Hosting)
